Project guide

This section explains what changed and why.

What got fixed

  • Removed the all-in-one giant HTML approach and split code into data, styles and app logic.
  • Added a real builds.json and expanded items.json with gear pieces used by the sample builds.
  • Added source labels so AI builds stop pretending to be verified meta.
  • Switched the PWA files to relative paths so GitHub uploads are less fragile.

What still needs real content

  • More builds. Right now the foundation is stronger than the content volume.
  • A larger item database if you want serious explorer depth.
  • Human review on top AI builds before treating them as meta recommendations.

Best next move

  • Keep adding real item entries, not more empty navigation.
  • Add a create/edit build form only after the item schema is stable.
  • Use this as the clean base for GitHub Pages.
